# Teste_Pi

Repositório de treinamento organizado em branches fixas:

- `main`: integração final
- `back-end`: código do backend
- `front-end`: código do frontend
- `docs`: documentação

Fluxo recomendado:

1. Trabalhar só na branch do componente.
2. Manter a branch sincronizada com `main` com frequência.
3. Usar worktrees locais ou checkouts separados para evitar misturar arquivos entre branches.
4. Fazer PR e merge apenas quando o bloco estiver pronto.
